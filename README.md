# 🤖 ربات تکلیف تلگرام

## نقش‌ها

| نقش | توضیح |
|-----|-------|
| 🎓 دانش‌آموز | پشتیبان انتخاب می‌کنه و تکلیف می‌فرسته |
| 👨‍🏫 پشتیبان | تکالیف شاگردهای خودش رو می‌بینه و جواب می‌ده |
| 👁️ ناظر کل | همه تکالیف همه بچه‌ها رو می‌بینه |

---

## دستورات ربات

### دانش‌آموز
- `/start` → انتخاب نقش + انتخاب پشتیبان
- `/send` → ارسال تکلیف جدید (بعد از انتخاب پشتیبان)

### پشتیبان
- `/start` → ثبت به عنوان پشتیبان
- `/my_homeworks` → لیست تکالیف دریافتی
- `/reply [شماره] [جواب]` → جواب به تکلیف

### ناظر کل
- `/all_homeworks` → دیدن همه تکالیف همه بچه‌ها

---

## راه‌اندازی روی Render.com

### ۱. ساخت ربات تلگرام
1. به [@BotFather](https://t.me/BotFather) بزن
2. `/newbot` بزن و اسم انتخاب کن
3. توکن رو کپی کن

### ۲. گرفتن ID ناظر کل
1. به [@userinfobot](https://t.me/userinfobot) پیام بده
2. ID عددی خودت رو کپی کن
3. اگه چند نفر ناظر هستن، IDها رو با کاما جدا کن: `123456,789012`

### ۳. آپلود روی GitHub
```
telegram-bot/
├── bot.py
├── database.py
├── requirements.txt
└── render.yaml
```
این ۴ فایل رو توی یه ریپوی GitHub آپلود کن.

### ۴. ساخت سرویس روی Render
1. وارد [render.com](https://render.com) بشو
2. **New** → **Blueprint** رو انتخاب کن
3. ریپوی GitHub رو وصل کن
4. Render خودکار از `render.yaml` می‌خونه

### ۵. تنظیم Environment Variables
توی Render، بخش **Environment**:

| نام | مقدار |
|-----|-------|
| `BOT_TOKEN` | توکنی که از BotFather گرفتی |
| `SUPERVISOR_IDS` | ID ناظر کل (مثلاً `123456789`) |
| `DB_PATH` | `/data/bot_data.db` |

### ۶. Deploy کن!
Render خودکار ربات رو راه‌اندازی می‌کنه.

## جریان کار

```
دانش‌آموز /start
    ↓
انتخاب پشتیبان
    ↓
ارسال تکلیف
    ↓
├── پشتیبان دریافت می‌کنه (/my_homeworks)
│       ↓
│   /reply [شماره] [جواب]
│       ↓
│   دانش‌آموز جواب دریافت می‌کنه
│
└── ناظر کل هم خودکار کپی دریافت می‌کنه (/all_homeworks)
```

---

## نکات مهم

- **پشتیبان‌ها باید اول `/start` بزنن** تا ثبت بشن، وگرنه دانش‌آموزها نمی‌تونن انتخابشون کنن
- **ناظر کل** باید ID عددیش توی `SUPERVISOR_IDS` باشه
- دیتابیس روی Disk ماندگار Render ذخیره میشه (1GB رایگان)
